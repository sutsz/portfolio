PopDaily - 提升搜尋轉換率

專案背景

什麼是 PopDaily？

PopDaily 是以 20-40 歲女性為受眾的內容平台，我負責提升網頁版搜尋轉換率，以數據驅動的方式迭代優化產品。其中解決方案包含排解正向流程的阻礙、與數據科學家協作調整演算法、調整 UI 畫面等。

目標

提升搜尋功能轉換率

在 PopDaily 網頁平台，搜尋功能使用深度不足，主要問題：搜尋結果頁 UI/UX 體驗不佳，且流程中存在摩擦點。目標是讓整體轉換率從 60% 顯著提升。

我的角色與具體工作內容

我的角色

負責迭代搜尋轉換方式的產品經理

具體工作內容

1. 分析搜尋 API 與搜尋紀錄

透過搜尋平台的成效報告與 API 數據，盤點熱門關鍵字與使用者的搜尋行為，找出搜尋結果頁的主要流失點。優先針對前 10 大熱門關鍵字進行優化，將搜尋成功率從 64.72% 提升至 69.38%。

2. 重新設計搜尋結果頁 UI

與設計師協作，重新設計行動版網頁（Mobile Web）的搜尋結果頁，優化結果的排序方式與卡片、文字呈現，降低使用者瀏覽與點擊的摩擦。

3. 提升搜尋結果的相關性

優化搜尋結果的排序與呈現邏輯，讓使用者更快找到符合需求的內容，並透過相關內容推薦延伸瀏覽，減少搜尋無結果或找不到目標的情況。

4. 與數據部門協作，調整排序演算法

與數據科學家合作測試 3 種排序演算法（v1／v2／v3），分別以「最新文章」、「30 天文章熱度（Gaussian Decay）」、「30 天文章熱度（Exponential Decay）」為基準，評估後決策上線 v3（Exponential Decay）。

搜尋結果排序優化前後：APP 與 Mobile Web 各週轉換率

關鍵成果與影響指標

搜尋結果點擊率（CTR）由 64.7% 提升至 71.5%

使用者操作步驟減少，能更快找到目標內容，搜尋體驗更直覺

搜尋結果頁的優化也帶動 SEO 流量成長，約佔整體 SEO 流量的 20%

遇到的挑戰與解決方式

既有排序演算法不適用

既有的排序模型在面對長尾關鍵字時表現不佳。若要讓搜尋成效達標，必須透過嚴謹的 A/B testing，先以小流量驗證，確認效果後再逐步放大到全量用戶。

如何解決：

與數據科學家密切合作，設計嚴謹的實驗框架，以 Exponential Decay 排序演算法取代舊有模型，並分批上線、逐步驗證成效。
